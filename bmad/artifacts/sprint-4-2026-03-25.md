# Sprint 4 — Agents built-in: WebSearch + PageFetch
Date: 2026-03-25

## Goal
Users can invoke `/search` and `/fetch` to trigger server-side agents that inject results into the conversation context before Ollama responds. Agent execution state is visible in the UI.

## Stories

| ID | Title | Effort | Depends on |
|----|-------|--------|------------|
| S4-01 | WebSearchAgent — server-side implementation | M | — |
| S4-02 | PageFetchAgent — server-side implementation | M | — |
| S4-03 | tool_calls DB + agent status API (SSE/polling) | M | S4-01, S4-02 |
| S4-04 | Frontend: agent status display in chat (spinner → result) | L | S4-03 |

## Capacity
Estimated: 4 stories × avg M = ~2–3 dev sessions

## Dependencies
- S4-01 and S4-02 are independent and can be built in parallel
- S4-03 requires both agents to wire tracing
- S4-04 requires S4-03 for the status API

## Acceptance (Sprint level)
- `/search <query>` triggers WebSearchAgent, results injected as `role: tool` message before Ollama call
- `/fetch <url>` triggers PageFetchAgent, page content injected similarly
- `tool_calls` table records input, output, status, timestamps
- Chat UI shows spinner while agent runs, result inline when done

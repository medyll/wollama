# Sprint 5 — Frontend UX: ToolCallMessage integration, HookInspector, Settings
Date: 2026-03-25

## Goal
Make agent results and hook activity visible in the chat UI. Wire ToolCallMessage into the message list, add a HookInspector panel for debugging hooks, and add a Settings page for managing skills/hooks.

## Stories

| ID | Title | Effort | Depends on |
|----|-------|--------|------------|
| S5-01 | Wire ToolCallMessage into chat message list | S | S4-04 done |
| S5-02 | HookInspector panel (debug hook_log per message) | M | — |
| S5-03 | Settings page — Skills management UI | M | — |
| S5-04 | Settings page — Hooks management UI | M | S5-03 |

## Capacity
4 stories, mix of S/M effort — ~2 dev sessions

## Dependencies
- S5-01 is independent (just wires existing component)
- S5-02 is independent (reads hook_log from message)
- S5-03 and S5-04 share the Settings page scaffold

## Acceptance (Sprint level)
- Chat messages with type tool_call render ToolCallMessage (spinner → result)
- hook_log on any message can be inspected in a collapsible panel
- Skills can be viewed, enabled/disabled from Settings
- Hooks can be viewed, enabled/disabled from Settings

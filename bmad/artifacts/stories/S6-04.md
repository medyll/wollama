# S6-04 — Polish: Loading states, error boundaries, toasts

## Goal
Improve UX with consistent loading states, proper error handling, and toast notifications for user feedback.

## Tasks
- Review current loading states across the app
- Add skeleton loaders for:
  - Chat list (sidebar)
  - Message streaming
  - Settings page (skills/hooks lists)
- Implement error boundaries:
  - Chat load failure (retry button)
  - Ollama connection error (user-friendly message)
  - Audio recording failure (fallback message)
- Enhance ToastContainer usage:
  - Copy to clipboard → success toast
  - Skill/hook toggle → confirmation toast
  - Error toasts with dismiss action
- Add global error boundary component

## Acceptance
- All major loading states have skeleton loaders
- Errors show user-friendly messages with recovery options
- Toast notifications appear for all user actions (success/error)
- No unhandled console errors

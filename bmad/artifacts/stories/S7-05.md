# S7-05 — Performance optimization

## Goal
Reduce bundle size and improve load times through code splitting and lazy loading.

## Tasks
**Bundle Analysis:**
- [ ] Run `npm run build` with bundle analyzer
- [ ] Identify largest dependencies
- [ ] Find code splitting opportunities

**Optimization:**
- [ ] Lazy load heavy components (CompanionSelector, Settings)
- [ ] Lazy load routes (onboarding, settings)
- [ ] Tree-shake unused Svelte components
- [ ] Optimize image assets (compress, WebP format)
- [ ] Remove unused dependencies

**Caching:**
- [ ] Implement service worker for offline caching
- [ ] Cache static assets with proper TTL
- [ ] Cache API responses where appropriate

**Performance Metrics:**
- [ ] Measure LCP (Largest Contentful Paint)
- [ ] Measure FID (First Input Delay)
- [ ] Measure CLS (Cumulative Layout Shift)
- [ ] Target: LCP < 2.5s, FID < 100ms, CLS < 0.1

## Acceptance
- Bundle size reduced by 10%+
- Lighthouse score > 90 on desktop
- Lighthouse score > 80 on mobile
- No regression in functionality

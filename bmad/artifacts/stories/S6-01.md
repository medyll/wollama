# S6-01 — Unit tests: Backend services (ollama, stt, tts)

## Goal
Add comprehensive unit tests for core backend services to ensure reliability and enable safe refactoring.

## Tasks
- Review existing service files in `server/services/`
- Create test files: `ollama.service.test.ts`, `stt.service.test.ts`, `tts.service.test.ts`
- Mock external dependencies (Ollama API, file system, audio binaries)
- Test key functions:
  - `ollama.service.ts`: generateResponse(), streamResponse(), listModels()
  - `stt.service.ts`: transcribeAudio(), validateAudioFormat()
  - `tts.service.ts`: generateSpeech(), selectVoice(), validateMood()
- Run tests with `npm run test:server -- --run`
- Aim for >70% coverage on tested services

## Acceptance
- All three test files exist and run with Vitest
- Tests cover happy path and error cases
- No tests rely on external services (all mocked)
- Tests pass consistently (no flakiness)

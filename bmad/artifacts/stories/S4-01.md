S4-01 — WebSearchAgent

Goal
Implement a server-side WebSearchAgent that takes a query string, fetches web search results, and returns structured data to be injected as a `role: tool` message in the conversation.

Tasks
- Create server/agents/web-search.agent.ts
- Agent accepts { query: string } input
- Fetches results via a configurable search provider (DuckDuckGo instant answer API or similar — no API key required for v1)
- Returns { results: [{ title, url, snippet }], query } shape
- Register agent in server/agents/index.ts with slug `web-search`
- Unit tests: mock fetch, verify output shape and error handling

Acceptance
- Agent returns ≥1 result for a valid query
- Empty/invalid query returns structured error (not a throw)
- Tests cover: valid query, empty query, network failure (mocked)

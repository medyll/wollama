S4-02 — PageFetchAgent

Goal
Implement a server-side PageFetchAgent that fetches a URL, extracts readable text content, and returns it for injection into the conversation as a `role: tool` message.

Tasks
- Create server/agents/page-fetch.agent.ts
- Agent accepts { url: string } input
- Fetches page with node fetch, strips HTML to readable text (use existing deps or a small utility — no new heavy deps)
- Truncates output to 4000 chars with a trailing note if truncated
- Returns { url, content, truncated: boolean } shape
- Register agent in server/agents/index.ts with slug `page-fetch`
- Unit tests: mock fetch, verify extraction, truncation, invalid URL handling

Acceptance
- Agent returns readable text for a valid public URL
- Unreachable/invalid URL returns structured error
- Content over 4000 chars is truncated with a note
- Tests cover: valid URL, invalid URL, oversized content

S5-04 — Settings page: Hooks management UI

Goal
Extend the Settings page with a Hooks tab/section where users can view all hooks and toggle them enabled/disabled.

Tasks
- Add Hooks section to the settings page (tab or accordion alongside Skills)
- Fetch GET /api/hooks on mount — add this route to server/routes/hooks.ts (new file)
- Display each hook: name, event, handler_type, priority, scope, enabled toggle
- Toggle calls PATCH /api/hooks/:id with { is_enabled: boolean }
- Unit tests: list renders, toggle calls endpoint

Acceptance
- Hooks list renders with event, priority, scope
- Toggle enables/disables a hook via API
- Tests cover: render list, toggle on, toggle off

S5-02 — HookInspector panel

Goal
Add a collapsible HookInspector panel to each chat message that has a non-empty hook_log, allowing developers to inspect which hooks ran, their duration, and any errors.

Tasks
- Create client/src/lib/components/HookInspector.svelte (Svelte 5 runes)
- Props: hook_log (array of { hook_id, event, duration_ms, mutated, error })
- Renders a collapsible <details> panel showing a table of hook entries
- Only visible when hook_log has at least 1 entry
- Shows: hook_id, event, duration_ms, mutated (badge), error (red if present)
- Unit tests: renders correctly with entries, hidden when empty, error row highlighted

Acceptance
- Panel hidden when hook_log is empty or absent
- Panel visible and collapsible when hook_log has entries
- Error entries visually distinct
- Tests cover: empty log, single entry, entry with error

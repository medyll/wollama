S4-04 — Frontend: agent status display in chat

Goal
Show agent execution state inline in the chat message list: a spinner while the agent runs, and the result (or error) once done.

Tasks
- Create client/src/lib/components/ToolCallMessage.svelte (Svelte 5 runes)
  - Props: tool_call_id, slug, input
  - Polls GET /api/agents/:tool_call_id/status every 1s until status !== 'running'
  - Shows: spinner (running), result content (done), error message (error)
- Integrate ToolCallMessage in the chat message list: when a message has type 'tool_call', render this component instead of the normal bubble
- Unit tests: spinner shown on running, result shown on done, error shown on error state

Acceptance
- Spinner visible while agent is running
- Result text visible and formatted after completion
- Error message visible if agent fails
- Polling stops once terminal state reached (done/error)
- Tests cover all three states

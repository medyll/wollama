# S6-02 — Unit tests: Frontend components (ChatWindow, MessageBubble, ToolCallMessage)

## Goal
Add unit tests for key Svelte 5 components to ensure UI reliability and prevent regressions.

## Tasks
- Review existing components in `client/src/components/`
- Create test files using Vitest + @testing-library/svelte:
  - `ChatWindow.test.svelte.ts`
  - `MessageBubble.test.svelte.ts`
  - `ToolCallMessage.test.svelte.ts`
- Mock Svelte 5 stores and RxDB dependencies
- Test key behaviors:
  - ChatWindow: renders messages, auto-scroll, empty state
  - MessageBubble: renders markdown, code highlighting, action buttons
  - ToolCallMessage: shows spinner during pending, displays result on done
- Run tests with `npm run test:client -- --run`

## Acceptance
- All three test files exist and run with Vitest + jsdom
- Tests cover rendering and key interactions
- No console errors during test runs
- Tests pass consistently

S5-03 — Settings page: Skills management UI

Goal
Add a Settings page (or section) where users can view all skills and toggle them enabled/disabled.

Tasks
- Create client/src/routes/settings/+page.svelte (or extend existing settings route if present)
- Fetch GET /api/skills on mount, display list with name, description, command, handler_type
- Toggle switch per skill calls PATCH /api/skills/:id with { is_enabled: boolean }
- Add PATCH /api/skills/:id route on server (server/routes/skills.ts)
- Unit tests: list renders, toggle calls correct endpoint

Acceptance
- Skills list renders with name, command, description
- Toggle enables/disables a skill via API
- Disabled skills visually distinct (greyed out)
- Tests cover: render list, toggle on, toggle off

# S6-03 — E2E smoke tests: Critical user flows

## Goal
Implement basic E2E smoke tests for critical user flows to catch major regressions before release.

## Tasks
- Review existing Playwright setup in `client/playwright.config.ts` and `client/e2e/`
- Create smoke test spec: `client/e2e/smoke.spec.ts`
- Test critical flows:
  1. **Send message**: Open chat, type message, send, see response stream
  2. **New chat**: Click "New Chat", verify empty chat view
  3. **Settings**: Navigate to settings, toggle a skill on/off
  4. **Hooks**: Navigate to settings, view hooks list, toggle a hook
- Use test fixtures or mock server to avoid Ollama dependency
- Run with `npx playwright test --project=chromium`

## Acceptance
- `smoke.spec.ts` exists with 4 test cases
- All tests pass on Chromium
- Tests are stable (no timing flakiness)
- Tests can run headless in CI
